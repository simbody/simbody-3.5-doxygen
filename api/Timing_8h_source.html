61  /* Posix nanosleep() sleeps the indicated number of nanoseconds and returns
62  0, or if it is interrupted early it returns how much time was left in
63  rem and returns EINTR. Ours is not interruptable so will always succeed and
64  return rem==0. It is OK if rem is NULL, but req==NULL or req<0 returns
65  EINVAL. A time of req==0 is allowed and our interpretation is that the
66  thread relinquishes its time slice to another ready-to-run thread if there
67  is one, otherwise returns immediately. This implementation rounds the
68  desired sleep time to the nearest millisecond. On a Linux system, this
69  requires including <time.h> (or <ctime>), which we already included
70  above. */
71  SimTK_SimTKCOMMON_EXPORT int nanosleep(const struct timespec* req, struct timespec* rem);

91  CLOCK_MONOTONIC are high resolution with no NTP adjustments. I measured
92  the resolutions on a single Windows 7 machine; hopefully they are typical
93  (resolution here means how often they are updated):
94  - MONOTONIC (counter): 0.001ms 1us
